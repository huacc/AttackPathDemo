## 一、系统概述

### 1.1 系统定位

本系统是一套面向网络空间安全领域的攻防仿真推演平台，基于本体建模、知识图谱、智能推演等技术，实现对HJZB网络环境的全方位建模、攻防态势推演、防御策略评估等核心功能。

### 1.2 核心能力

- **网络空间本体建模**: 构建网络资产、攻击行为、防御措施的本体模型体系
- **知识图谱构建**: 基于ATT&CK、D3FEND框架的攻防知识图谱
- **攻防场景推演**: 多算法驱动的攻击路径推演与防御策略仿真
- **态势可视化**: 2D网络沙盘、攻击链路、损伤评估的实时可视化
- **智能决策支持**: 基于推演结果的防御策略优化建议

### 1.3 技术架构

- **前端框架**: React 18 + TypeScript + Ant Design 5.x
- **图形引擎**: AntV G6 (图谱 + 2D沙盘)
- **状态管理**: Zustand / Redux Toolkit
- **数据可视化**: ECharts + AntV G2

## 三、核心功能详细设计

### 模块1: 攻防场景管理模块

#### 1.1 场景配置中心

**功能描述**
提供攻防场景的全生命周期管理，支持场景创建、编辑、克隆、删除、导入导出等操作。

**页面布局**
- 左侧：场景树形列表（支持分级分类）
- 中间：场景详情配置区
- 右侧：场景预览缩略图

**核心交互**

1. **场景创建流程**
   - 步骤1：选择场景模板（空白/预设模板）
   - 步骤2：配置基础信息（名称、描述、标签、分类）
   - 步骤3：选择资产范围（从资产库导入或手动添加）
   - 步骤4：配置业务参数（业务系统、依赖关系）
   - 步骤5：设置推演参数（时间范围、推演速度、算法选择）

2. **场景编辑操作**
   - 拖拽式资产添加到场景画布
   - 右键菜单：编辑属性、删除、复制、锁定
   - 批量操作：框选多个实体进行批量编辑
   - 实时保存：自动保存草稿，支持版本回退

3. **场景数据管理**
   - 导出格式：JSON、XML、自定义格式
   - 导入校验：数据完整性检查、冲突处理
   - 离线加载：支持无网络环境下加载场景数据

**数据逻辑**

#### 1.2 资产拓扑建模

**功能描述**
基于实网采集数据，构建网络资产的拓扑关系，支持多层级（业务层、网络层、物理层）的拓扑展示与编辑。

**页面布局**
- 顶部：工具栏（布局切换、图层控制、缩放、搜索）
- 中间：拓扑画布（支持拖拽、缩放、平移）
- 右侧：属性面板（选中节点/边的详细信息）
- 底部：图层控制器（业务层/网络层/攻防层切换）

**核心交互**

1. **拓扑构建**
   - 自动布局：根据采集数据自动生成拓扑
   - 手动调整：拖拽节点调整位置
   - 连线编辑：添加/删除/编辑连接关系
   - 分组管理：按业务/区域对资产进行分组

2. **图层控制**
   - 业务层：展示业务系统及其依赖关系
   - 网络层：展示网络设备、链路、流量
   - 攻防层：展示攻击路径、防御设施
   - 图层叠加：支持多图层同时显示，透明度可调

3. **拓扑分析**
   - 路径分析：计算两点间的所有可达路径
   - 关键节点识别：基于度中心性、介数中心性算法
   - 脆弱性热力图：根据CVE数量着色显示
   - 流量可视化：实时流量动画展示

#### 1.3 业务依赖分析

**功能描述**
分析业务系统之间的依赖关系，计算业务依赖度，识别关键业务路径和单点故障风险。

**页面布局**
- 左侧：业务系统列表
- 中间：业务依赖关系图
- 右侧：依赖分析结果面板

**核心交互**

1. **依赖关系建模**
   - 手动添加：选择源业务和目标业务，定义依赖类型
   - 自动推导：基于流量分析自动发现依赖关系
   - 依赖强度：设置依赖强度（强依赖/弱依赖/可选依赖）
   - 依赖条件：配置依赖生效的条件（时间、状态等）

2. **依赖度计算**
   - 直接依赖度：A直接依赖B的程度
   - 间接依赖度：A通过中间节点依赖B的程度
   - 综合依赖度：考虑所有路径的综合依赖评分
   - 依赖链路：展示完整的依赖传递链

3. **风险识别**
   - 单点故障分析：识别关键节点，评估其失效影响
   - 依赖环检测：检测循环依赖并提示风险
   - 关键路径标注：高亮显示关键业务路径
   - 影响范围评估：模拟某节点失效的影响范围

**数据逻辑**

- 连通性分析：评估业务隔离性

**关键UI组件**

### 模块2: 本体建模模块

#### 2.1 本体设计器

**功能描述**
提供可视化的本体建模工具，支持实体类、属性、关系的定义和编辑，参考demo中的InteractiveOntologyModeling组件。

**页面布局**
- 左侧：本体元素库（实体类模板、关系模板）
- 中间：建模画布（拖拽式建模）
- 右侧：属性配置面板
- 底部：本体验证和导出工具栏

**核心交互**

1. **实体类建模**
   - 拖拽创建：从左侧库拖拽实体类模板到画布
   - 属性定义：添加/编辑实体类的属性（名称、类型、约束）
   - 继承关系：通过连线定义实体类的继承关系
   - 实例化预览：查看实体类的实例化示例

2. **关系建模**
   - 关系定义：定义实体类之间的关系（关联、聚合、组合）
   - 基数约束：设置关系的基数（1:1, 1:N, N:M）
   - 关系属性：为关系添加属性（如权重、置信度）
   - 反向关系：自动生成反向关系

3. **本体验证**
   - 一致性检查：检查本体定义的逻辑一致性
   - 完整性检查：检查必需属性是否定义
   - 冲突检测：检测命名冲突、循环继承等问题
   - 推理验证：基于OWL推理机验证本体

   - 约束条件：设置路径约束（最大跳数、时间限制）

## 四、关键交互流程

### 4.1 完整推演流程

**流程描述**
从场景配置到推演执行，再到结果分析的完整闭环流程。

**流程步骤**

1. **场景准备阶段**
   - 用户进入"攻防场景管理"模块
   - 创建新场景或选择已有场景
   - 配置场景参数（资产、业务、网络拓扑）
   - 导入实网数据或手动添加实体
   - 保存场景配置

2. **推演配置阶段**
   - 进入"攻防驾驶舱"模块
   - 选择推演类型（攻击路径/防御策略/综合推演）
   - 配置推演参数（起点、目标、算法、约束）
   - 选择评估算法和指标
   - 预览推演配置

3. **推演执行阶段**
   - 启动推演引擎
   - 实时展示推演过程（2D沙盘动画）
   - 显示中间结果和关键事件
   - 支持暂停、继续、停止操作
   - 记录推演日志

4. **结果分析阶段**
   - 展示推演结果（攻击路径、防御效果）
   - 进行损伤评估和影响分析
   - 生成评估报告和建议
   - 支持结果对比和历史回溯
   - 导出分析报告

**数据流转**

```
场景数据 → 推演引擎 → 算法库 → 本体模型 → 知识图谱 → 评估结果 → 报告生成
    ↓           ↓          ↓         ↓          ↓          ↓          ↓
  资产库    实时监控    参数配置   推理引擎   可视化展示  损伤评估   历史记录
```

---

### 4.2 本体建模到知识图谱流程

**流程描述**
从本体设计到知识实例化，再到知识推理的完整流程。

**流程步骤**

1. **本体设计**
   - 进入"本体建模"模块
   - 使用本体设计器创建实体类和关系
   - 定义属性和约束
   - 验证本体一致性
   - 保存本体模型

2. **实体建模**
   - 进入"实体模型库"
   - 基于本体创建具体实体
   - 配置实体属性和量化指标
   - 建立实体间的关系
   - 映射到ATT&CK/D3FEND框架

3. **知识实例化**
   - 进入"知识图谱"模块
   - 基于实体模型生成知识实例
   - 导入外部知识（威胁情报）
   - 建立知识关联关系
   - 设置置信度和来源

4. **知识推理**
   - 配置推理任务
   - 选择推理规则
   - 执行推理引擎
   - 发现新知识和关系
   - 更新知识图谱

**数据流转**

```
本体模型 → 实体模板 → 实体实例 → 知识节点 → 推理引擎 → 新知识
    ↓          ↓          ↓          ↓          ↓          ↓
  验证器    属性配置    关系建立    图谱展示    规则库    图谱更新
```

### 4.3 攻击路径推演交互流程

**流程描述**
配置攻击参数、执行路径推演、分析结果的详细交互流程。

**交互步骤**

1. **配置攻击起点**
   - 在2D沙盘画布中选择起点节点
   - 或从资产列表中选择起点
   - 配置攻击者画像（技能、资源、动机）
   - 设置已知漏洞和工具

2. **配置攻击目标**
   - 选择目标节点或目标类型
   - 设置攻击目标（数据窃取、服务中断、权限提升）
   - 配置目标优先级

3. **配置推演参数**
   - 选择推演算法（随机游走/贝叶斯/BFS/DFS）
   - 设置约束条件（最大跳数、时间限制、隐蔽性要求）
   - 配置推演深度和广度
   - 选择评估指标

4. **执行推演**
   - 点击"开始推演"按钮
   - 实时展示推演进度
   - 在沙盘中动画展示攻击路径
   - 显示每一跳的成功率和检测率

5. **分析结果**
   - 查看所有发现的攻击路径

### 5.2 图谱渲染优化

**性能优化策略**

1. **大规模图谱渲染**
   - 虚拟化渲染：只渲染可视区域的节点
   - LOD（Level of Detail）：根据缩放级别调整细节
   - 节点聚合：自动聚合密集区域的节点
   - 增量渲染：增量更新而非全量重绘

2. **交互性能优化**
   - 防抖和节流：优化频繁触发的事件
   - Web Worker：将计算密集任务移到Worker
   - 懒加载：按需加载节点详情数据
   - 缓存策略：缓存布局计算结果

3. **内存管理**
   - 及时销毁：组件卸载时销毁图谱实例
   - 对象池：复用节点和边对象
   - 图片压缩：压缩节点图标资源
   - 内存监控：监控内存使用情况

### 5.3 2D沙盘实现

**技术选型**

**核心功能实现**

   - 设置分层布局（安全域背景色块 + 节点层 + 边层 + 动画层）

   - 框选多节点、右键菜单

   - 节点聚合：超过 500 节点时启用 combo 分组折叠
