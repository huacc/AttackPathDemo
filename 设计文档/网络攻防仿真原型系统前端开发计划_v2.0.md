## 一、项目概述

### 1.1 改造目标

将现有心理分析系统原型（demo/）端到端改造为网络攻防仿真推演系统前端原型，满足《原始需求》全部条款。

### 1.2 改造策略

- **就地改造**：直接在 `demo/` 目录上修改，不新建目录
- **去除 Gemini**：移除 AI 聊天功能，系统完全由 Mock 数据驱动
- **统一 G6**：所有图形可视化（沙盘、图谱、本体）统一使用 AntV G6，移除 ReactFlow
- **增量替换**：保留可复用的基础设施（Vite、Ant Design、Zustand、布局框架），替换业务层

### 1.3 项目边界

**在范围内**：
- 纯前端 Demo 原型，所有数据 Mock
- 覆盖《需求》7 大功能模块
- 满足《原始需求》(1)-(4) 全部条款
- 单用户模式，无登录权限

**不在范围内**：
- 真实后端 API / 算法计算 / 数据采集
- 完整 ATT&CK/D3FEND 全量数据（选取核心子集即可）
- 多用户、权限管理、生产级性能优化

### 1.4 技术选型

| 技术 | 版本 | 用途 | 备注 |
|------|------|------|------|

## 五、开发规范

### 5.1 命名规范

- 组件文件：PascalCase（`AttackPathSimulation.tsx`）
- 工具函数：camelCase（`formatCvssScore.ts`）
- Mock 数据文件：camelCase（`networkNodes.ts`）
- 类型定义：PascalCase（`NetworkNode`、`AttackTechnique`）
- 常量：UPPER_SNAKE_CASE（`MAX_NODE_COUNT`）
- 枚举值：snake_case（`nation_state`、`dmz`）— 与《本体规范》保持一致

### 5.2 Mock 数据规范

- 所有 Mock 数据使用 TypeScript 强类型，禁止 `as any`
- CVE/ATT&CK/D3FEND ID 必须使用真实存在的标识符
- 数值型属性（评分、概率）必须在合理范围内
- 跨实体引用必须使用已定义的 ID，禁止硬编码不存在的 ID
- 每个 Mock 数据文件顶部注释说明数据来源和数量

### 5.3 组件规范

- 使用函数式组件 + Hooks
- 复杂组件拆分为容器组件（数据逻辑）+ 展示组件（纯 UI）
- G6 相关组件统一封装 `useG6Graph` Hook 管理生命周期
- 详情面板统一使用 Ant Design Drawer 组件
