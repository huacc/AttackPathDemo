# 本体模型体系设计规范

> **文档版本**: v2.0
> **修订日期**: 2026-03-14
> **适用范围**: 网络攻防仿真原型系统 - 本体建模模块
> **参考标准**: STIX 2.1 / CybOX 2.1 / ATT&CK v14 / D3FEND v1.0 / CVSS v3.1

---

## 一、设计原则

### 1.1 抽象归一原则

**核心思想**：同一类事物抽象为一个本体，用枚举属性区分具体类型，而非每种类型建一个本体。

```
❌ 错误做法：Server本体 + Router本体 + Firewall本体 + Switch本体...
✅ 正确做法：NetworkNode本体（deviceType枚举区分 Server/Router/Firewall/Switch）

❌ 错误做法：SQLInjection本体 + XSS本体 + RCE本体...
✅ 正确做法：AttackTechnique本体（techniqueId映射ATT&CK，attackCategory枚举区分）
```

### 1.2 领域语义原则

每个本体的属性必须具备**网络安全领域的语义**，不能是通用属性。判断标准：
- 属性值必须来自网络安全标准（CVE/CVSS/CPE/ATT&CK ID/D3FEND ID等）
- 属性必须能支撑攻击路径推演、防御效果评估等仿真计算
- 属性必须能区分该本体与其他领域同类事物（如区分网络设备与汽车ECU）

### 1.3 界面可见性原则

每个属性标注三种界面状态：
- `[显示]`：在沙盘/列表中直接可见
- `[详情]`：点击节点后在详情面板可见
- `[编辑]`：用户可在界面上修改

## 二、本体模型总览

系统共定义 **6个核心本体**，覆盖网络攻防仿真的全部领域对象：

| 本体名称 | 英文标识 | 参考标准 | 抽象范围 | 实例数量 |
|---------|---------|---------|---------|---------|
| 网络节点本体 | NetworkNode | CybOX NetworkInterface / STIX Infrastructure | 所有硬件+软件资产 | 20-30个 |
| 漏洞本体 | Vulnerability | CVE/CVSS/CWE/NVD | 所有软硬件漏洞 | 10-20个 |
| 攻击技术本体 | AttackTechnique | ATT&CK v14 Technique | 所有攻击手法 | 30-50个 |
| 威胁行为者本体 | ThreatActor | STIX 2.1 threat-actor | 攻击者/组织 | 3-5个 |
| 防御措施本体 | DefenseMeasure | D3FEND v1.0 / STIX course-of-action | 所有防御手段 | 15-25个 |
| 攻防关系本体 | AttackDefenseRelation | ATT&CK Mitigation映射 | 攻防映射关系 | 50+条 |

## 三、NetworkNode 网络节点本体

### 3.1 本体定义

**抽象范围**：网络空间中所有可寻址的实体节点，包括物理硬件设备和运行在其上的软件服务。一个物理服务器是一个 NetworkNode，运行在其上的 Web 应用也是一个 NetworkNode（通过 `hostedOn` 关系关联）。

**参考标准**：
- CybOX 2.1 `NetworkInterface Object` / `System Object`
- STIX 2.1 `infrastructure` / `software`
- CPE（Common Platform Enumeration）命名规范

### 3.2 属性定义

#### 标识维度（Identity）

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `nodeId` | string | [显示] | 节点唯一标识，格式：`node-{uuid}` | 内部 |
| `name` | string | [显示][编辑] | 节点名称，如 `web-server-01` | 内部 |
| `nodeType` | enum | [显示][编辑] | 节点类型：`hardware` / `software` | 内部 |
| `deviceCategory` | enum | [显示][编辑] | 设备分类（见下方枚举表） | CybOX System Object |
| `cpe` | string | [详情] | CPE 2.3 标准命名，如 `cpe:2.3:a:nginx:nginx:1.18.0:*:*:*:*:*:*:*` | NVD/CPE |
| `zone` | enum | [显示][编辑] | 所属安全域：`external`/`dmz`/`intranet`/`cloud` | 内部 |

**deviceCategory 枚举值**：

| 枚举值 | 含义 | 典型实例 |
|-------|------|---------|
| `server` | 通用服务器 | Web服务器、应用服务器 |
| `router` | 路由器 | 边界路由器、核心路由器 |
| `switch` | 交换机 | 接入层交换机、汇聚层交换机 |
| `firewall` | 防火墙 | 边界防火墙、内网防火墙 |
| `load_balancer` | 负载均衡器 | F5、Nginx LB |
| `endpoint` | 终端设备 | 工作站、笔记本 |
| `iot_device` | 物联网设备 | 工控设备、传感器 |
| `web_application` | Web应用 | CMS、OA系统 |
| `database` | 数据库服务 | MySQL、Oracle |
| `middleware` | 中间件 | Tomcat、Redis、Kafka |
| `container` | 容器/虚拟机 | Docker容器、KVM虚拟机 |
| `api_gateway` | API网关 | Kong、Nginx API |
| `domain_controller` | 域控制器 | Windows AD DC |

#### 网络标识维度（Network Identity）— 参考 CybOX NetworkInterface

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `ipv4Address` | string | [显示][编辑] | IPv4地址，CIDR格式，如 `192.168.1.10/24` | RFC 4632 |
| `ipv6Address` | string | [详情][编辑] | IPv6地址 | RFC 4291 |
| `macAddress` | string | [详情] | MAC地址，格式 `AA:BB:CC:DD:EE:FF` | IEEE 802 |
| `hostname` | string | [显示][编辑] | FQDN主机名，如 `web01.corp.local` | RFC 1123 |
| `vlanId` | number | [详情][编辑] | VLAN ID，范围 1-4094 | IEEE 802.1Q |
| `asn` | number | [详情] | 自治系统号（外网节点适用） | RFC 1930 |
| `openPorts` | array | [详情][编辑] | 开放端口列表（见 NetworkService 结构） | IANA Port Numbers |

**NetworkService 结构**（openPorts 数组元素）：

```typescript
interface NetworkService {
  port: number;           // 端口号 1-65535
  protocol: 'tcp'|'udp'; // 传输层协议
  service: string;        // 服务名，如 'http','ssh','smb'
  version: string;        // 服务版本，如 'nginx/1.18.0'
  banner: string;         // 服务 banner（可选）
  state: 'open'|'filtered'|'closed';
}
```

#### 系统指纹维度（System Fingerprint）— 参考 CybOX System Object

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `osType` | enum | [显示][编辑] | 操作系统类型：`windows`/`linux`/`macos`/`ios`/`android`/`embedded` | CybOX OS |
| `osVersion` | string | [详情][编辑] | OS版本，如 `Ubuntu 20.04.3 LTS` | CPE |
| `osCpe` | string | [详情] | OS的CPE标识，如 `cpe:2.3:o:canonical:ubuntu_linux:20.04:*` | NVD/CPE |
| `kernelVersion` | string | [详情] | 内核版本，如 `5.4.0-150-generic` | 内部 |
| `architecture` | enum | [详情] | CPU架构：`x86_64`/`arm64`/`mips`/`ppc` | 内部 |
| `patchLevel` | enum | [显示] | 补丁状态：`up_to_date`/`outdated`/`eol`/`unknown` | 内部 |
| `lastPatchDate` | date | [详情] | 最后打补丁日期 | 内部 |

#### 安全态势维度（Security Posture）— 核心仿真维度

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `attackSurface` | number | [显示] | 攻击面评分 0-100，由开放端口数×漏洞数计算 | 内部计算 |
| `exploitabilityScore` | number | [显示] | 可利用性评分 0-10，聚合所有漏洞的 CVSS ExploitabilityScore | CVSS v3.1 |
| `criticalityLevel` | enum | [显示][编辑] | 资产重要性：`critical`/`high`/`medium`/`low` | 内部 |
| `dataClassification` | enum | [详情][编辑] | 数据敏感级别：`top_secret`/`secret`/`confidential`/`public` | 内部 |
| `complianceStatus` | enum | [详情] | 安全基线合规状态：`compliant`/`non_compliant`/`partial`/`unknown` | 内部 |
| `securityScore` | number | [显示] | 综合安全评分 0-100（越高越安全） | 内部计算 |

#### 运行状态维度（Runtime State）

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `status` | enum | [显示] | 运行状态：`online`/`offline`/`degraded`/`compromised` | 内部 |
| `compromiseIndicators` | array | [详情] | 失陷指标列表（IOC），参考 STIX Indicator | STIX 2.1 |
| `lastSeenAt` | datetime | [详情] | 最后活跃时间 | 内部 |

## 四、Vulnerability 漏洞本体

### 4.1 本体定义

**抽象范围**：所有可被攻击者利用的软硬件缺陷，包括 CVE 已知漏洞、配置错误、协议弱点。一个漏洞本体实例对应一个具体的可利用缺陷，通过 `exists_in` 关系挂载到 NetworkNode 上。

**参考标准**：
- CVE（Common Vulnerabilities and Exposures）
- CVSS v3.1（Common Vulnerability Scoring System）
- CWE（Common Weakness Enumeration）
- EPSS（Exploit Prediction Scoring System）

### 4.2 属性定义

#### 漏洞标识维度（Vulnerability Identity）

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `vulnId` | string | [显示] | 内部唯一标识 `vuln-{uuid}` | 内部 |
| `cveId` | string | [显示] | CVE编号，如 `CVE-2023-44487` | NVD/CVE |
| `cweId` | string | [详情] | CWE弱点类型，如 `CWE-89`（SQL注入） | MITRE CWE |
| `title` | string | [显示] | 漏洞标题，如 `HTTP/2 Rapid Reset Attack` | NVD |
| `description` | string | [详情] | 漏洞详细描述 | NVD |
| `affectedCpe` | string | [详情] | 受影响软件的CPE标识 | NVD/CPE |
| `publishedDate` | date | [详情] | CVE发布日期 | NVD |
| `patchAvailable` | boolean | [显示] | 是否有官方补丁 | NVD |

#### CVSS v3.1 评分维度（Scoring）— 直接影响仿真计算

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `cvssScore` | number | [显示] | CVSS v3.1 基础评分 0.0-10.0 | CVSS v3.1 |
| `cvssVector` | string | [详情] | CVSS向量字符串，如 `CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H` | CVSS v3.1 |
| `attackVector` | enum | [显示] | 攻击向量：`Network`/`Adjacent`/`Local`/`Physical` | CVSS v3.1 AV |
| `attackComplexity` | enum | [显示] | 攻击复杂度：`Low`/`High` | CVSS v3.1 AC |
| `privilegesRequired` | enum | [详情] | 所需权限：`None`/`Low`/`High` | CVSS v3.1 PR |
| `userInteraction` | enum | [详情] | 用户交互：`None`/`Required` | CVSS v3.1 UI |
| `scope` | enum | [详情] | 影响范围：`Unchanged`/`Changed` | CVSS v3.1 S |
| `confidentialityImpact` | enum | [详情] | 机密性影响：`None`/`Low`/`High` | CVSS v3.1 C |
| `integrityImpact` | enum | [详情] | 完整性影响：`None`/`Low`/`High` | CVSS v3.1 I |
| `availabilityImpact` | enum | [详情] | 可用性影响：`None`/`Low`/`High` | CVSS v3.1 A |
| `exploitabilityScore` | number | [显示] | CVSS可利用性子评分 0-3.9 | CVSS v3.1 计算 |
| `impactScore` | number | [显示] | CVSS影响子评分 0-6.0 | CVSS v3.1 计算 |

#### 利用情报维度（Exploit Intelligence）— 仿真推演核心

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `epssScore` | number | [显示] | EPSS利用预测评分 0.0-1.0（越高越可能被利用） | EPSS |
| `epssPercentile` | number | [详情] | EPSS百分位排名 | EPSS |
| `exploitMaturity` | enum | [显示][编辑] | 利用成熟度：`not_defined`/`unproven`/`poc`/`functional`/`high` | CVSS v3.1 E |
| `exploitAvailable` | boolean | [显示] | 是否有公开利用代码（EDB/Metasploit等） | 内部 |
| `exploitSource` | string | [详情] | 利用代码来源，如 `Metasploit:exploit/multi/handler` | 内部 |
| `kevListed` | boolean | [显示] | 是否在CISA KEV（已知被利用漏洞）目录中 | CISA KEV |
| `inTheWild` | boolean | [显示] | 是否有在野利用记录 | 内部 |

#### 修复状态维度（Remediation）

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `remediationStatus` | enum | [显示][编辑] | 修复状态：`unpatched`/`patched`/`mitigated`/`accepted_risk` | 内部 |
| `patchId` | string | [详情][编辑] | 补丁编号，如 `MS23-001`、`USN-6315-1` | 内部 |
| `workaround` | string | [详情][编辑] | 临时缓解措施描述 | 内部 |

## 五、AttackTechnique 攻击技术本体

### 5.1 本体定义

**抽象范围**：攻击者在攻击过程中使用的所有技术手法，直接映射 ATT&CK v14 的 Technique/Sub-technique 体系。一个 AttackTechnique 实例对应一个具体的攻击步骤，通过 `uses_technique` 关系挂载到攻击路径上。

**参考标准**：
- MITRE ATT&CK v14（Enterprise Matrix）
- STIX 2.1 `attack-pattern` 对象
- Cyber Kill Chain（Lockheed Martin）

### 5.2 属性定义

#### ATT&CK 标识维度（ATT&CK Identity）— 直接对齐标准

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `techniqueId` | string | [显示] | ATT&CK技术ID，如 `T1190`、`T1059.001` | ATT&CK |
| `techniqueName` | string | [显示] | 技术名称，如 `Exploit Public-Facing Application` | ATT&CK |
| `tacticId` | string | [显示] | 所属战术ID，如 `TA0001` | ATT&CK |
| `tacticName` | string | [显示] | 战术名称，如 `Initial Access` | ATT&CK |
| `isSubTechnique` | boolean | [详情] | 是否为子技术（如 T1059.001 是 T1059 的子技术） | ATT&CK |
| `parentTechniqueId` | string | [详情] | 父技术ID（子技术适用） | ATT&CK |
| `stixId` | string | [详情] | STIX 2.1 attack-pattern ID，如 `attack-pattern--{uuid}` | STIX 2.1 |
| `killChainPhase` | enum | [显示] | Kill Chain阶段：`reconnaissance`/`weaponization`/`delivery`/`exploitation`/`installation`/`command_and_control`/`actions_on_objectives` | Cyber Kill Chain |

#### 技术特征维度（Technique Characteristics）— 仿真推演核心

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `requiredPrivilege` | enum | [显示] | 执行所需权限：`none`/`user`/`admin`/`system`/`root` | ATT&CK |
| `requiredAccess` | enum | [显示] | 所需访问类型：`remote_network`/`local_network`/`local`/`physical` | ATT&CK |
| `targetPlatforms` | array | [显示] | 目标平台：`windows`/`linux`/`macos`/`cloud`/`network`/`containers` | ATT&CK |
| `dataSourcesDetectable` | array | [详情] | 可检测的数据源，如 `['Network Traffic','Process Creation']` | ATT&CK Data Sources |
| `defensesBypassed` | array | [详情] | 可绕过的防御措施，如 `['Antivirus','Firewall']` | ATT&CK |
| `permissionsRequired` | array | [详情] | 所需权限列表 | ATT&CK |

#### 仿真量化维度（Simulation Metrics）— 推演计算专用

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `baseSuccessRate` | number | [显示][编辑] | 基础成功率 0.0-1.0（无防御时） | 内部（可编辑） |
| `detectionDifficulty` | enum | [显示][编辑] | 检测难度：`easy`/`medium`/`hard`/`very_hard` | 内部 |
| `avgDwellTime` | number | [详情][编辑] | 平均驻留时间（秒），用于时间轴推演 | 内部 |
| `noiseLevel` | enum | [详情][编辑] | 噪声级别（被检测的可能性）：`low`/`medium`/`high` | 内部 |
| `lateralMovementCapable` | boolean | [详情] | 是否支持横向移动 | 内部 |

| 属性 | 类型 | 界面状态 | 说明 | 标准来源 |
|-----|------|---------|------|---------|
| `associatedTools` | array | [详情] | 关联工具列表，如 `['Metasploit','Cobalt Strike']` | ATT&CK Software |

## 八、AttackDefenseRelation 攻防关系本体

### 8.1 本体定义

**抽象范围**：攻击技术与防御措施之间的映射关系，以及攻击路径中节点间的攻击传播关系。这是一个**关系型本体**，实例本身就是关系记录，不代表独立实体。

**参考标准**：
- ATT&CK v14 Mitigation 映射表（Technique ↔ Mitigation）
- STIX 2.1 `relationship` 对象
- D3FEND Offensive-Defensive 映射

### 8.2 关系类型定义

系统中存在以下 **7 种核心关系**：

| 关系类型 | 源本体 | 目标本体 | 语义 | STIX关系类型 |
|---------|-------|---------|------|------------|
| `exists_in` | Vulnerability | NetworkNode | 漏洞存在于节点上 | `related-to` |
| `exploits` | AttackTechnique | Vulnerability | 攻击技术利用漏洞 | `targets` |
| `targets` | AttackTechnique | NetworkNode | 攻击技术直接针对节点 | `targets` |
| `attributed_to` | AttackPath | ThreatActor | 攻击路径归因于行为者 | `attributed-to` |
| `mitigates` | DefenseMeasure | AttackTechnique | 防御措施缓解攻击技术 | `mitigates` |
| `deployed_on` | DefenseMeasure | NetworkNode | 防御措施部署在节点上 | `related-to` |
| `propagates_to` | NetworkNode | NetworkNode | 攻击从源节点传播到目标节点 | `related-to` |

### 8.3 AttackDefenseRelation 实例属性

每条关系记录包含以下属性：

| 属性 | 类型 | 界面状态 | 说明 |
|-----|------|---------|------|
| `relationId` | string | [详情] | 关系唯一标识 `rel-{uuid}` |
| `relationType` | enum | [显示] | 关系类型（见上方7种） |
| `sourceId` | string | [详情] | 源对象 ID |
| `targetId` | string | [详情] | 目标对象 ID |
| `effectivenessScore` | number | [显示] | 关系效能评分 0-100（mitigates关系：防御效果；exploits关系：利用成功率） |
| `conditionExpression` | string | [详情] | 关系生效条件，如 `"remediationStatus == 'unpatched'"` |
| `validatedBy` | string | [详情] | 验证来源，如 `ATT&CK:M1050→T1190` |

### 8.4 攻防映射矩阵（核心数据）

系统预置 ATT&CK Mitigation 映射，格式示例：

```
T1190 (Exploit Public-Facing App)
  ← M1050 Exploit Protection        effectivenessScore: 80
  ← M1016 Vulnerability Scanning    effectivenessScore: 60
  ← M1026 Privileged Account Mgmt   effectivenessScore: 40

T1059.001 (PowerShell)
  ← M1045 Code Signing               effectivenessScore: 70
  ← M1042 Disable or Remove Feature  effectivenessScore: 90
  ← M1038 Execution Prevention       effectivenessScore: 75

T1071.001 (Web Protocols C2)
  ← M1031 Network Intrusion Prevention  effectivenessScore: 65
  ← M1037 Filter Network Traffic        effectivenessScore: 55
```

## 十一、本体设计检查清单

在实现本体模型时，逐项核对以下清单：

### 11.1 抽象归一检查

- [ ] 每类事物只有一个本体（不存在 Server本体 + Router本体 并列）
- [ ] 同类对象的区分通过枚举属性实现（deviceCategory / actorType 等）
- [ ] 没有为"特殊情况"单独创建本体（如不存在 APTActor 本体）

### 11.2 领域语义检查

- [ ] 每个本体至少有 3 个属性来自网络安全标准（CVE/CVSS/ATT&CK/D3FEND/CPE）
- [ ] 没有纯通用属性（如单独的 `description` 字段不算领域属性）
- [ ] 仿真计算所需的量化属性已定义（成功率/检测率/评分等）
- [ ] 属性值格式符合对应标准（CVE格式/CVSS向量格式/ATT&CK ID格式）

### 11.3 界面可见性检查

- [ ] 每个属性都标注了 [显示]/[详情]/[编辑] 状态
- [ ] [显示] 属性不超过 6 个（沙盘节点卡片空间有限）
- [ ] [编辑] 属性都有合理的输入验证规则
- [ ] 沙盘节点卡片 ASCII 图已绘制

### 11.4 关系完整性检查

- [ ] 7 种核心关系类型都已定义
- [ ] 每种关系都有 STIX 2.1 对应类型
- [ ] 关系的 effectivenessScore 有计算逻辑说明
- [ ] ATT&CK Mitigation 映射数据已预置（至少覆盖 20 个常用技术）

### 11.5 Mock 数据检查

- [ ] 每个本体至少有 1 个完整的 JSON 实例示例
- [ ] 实例中的 CVE/ATT&CK ID 是真实存在的（不是虚构的）
- [ ] CVSS 向量字符串格式正确（可用 NVD 计算器验证）
- [ ] 实例间的关系引用 ID 一致（nodeId 与 deployedNodes 中的值匹配）

### 11.6 仿真推演支撑检查

- [ ] 攻击路径推演所需属性已覆盖：
  - NetworkNode.exploitabilityScore（节点可利用性）
