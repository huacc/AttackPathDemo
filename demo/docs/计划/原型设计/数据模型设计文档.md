
# 心理分析系统 - 数据模型设计文档 (P1.1)

> **版本**: v1.0
> **状态**: 已完成
> **日期**: 2026-01-27
> **设计原则**: 原子化、可追溯、层级解耦

## 1. 概述
本文档定义了心理分析系统原型阶段的核心数据结构。设计遵循心理学原子化建模方法论，重点解决从“原始证据”到“分析结论”的逻辑链条闭环。

## 2. 核心实体关系 (ER)

### 2.1 证据追溯链 (Evidence Traceability)
结论的可靠性建立在完整的证据链上：
- **Evidence (证据)**: 最小信息单元，包含文本、图片片段。
- **ReasoningCondition (推理条件)**: 规则的组成部分，必须关联一个或多个证据。
- **ReasoningChain (推理链条)**: 整合规则、条件与证据，形成最终结论的推导逻辑。

### 2.2 对话与分析 (Conversation & Analysis)
- **Conversation**: 承载一次分析任务的容器。
- **Message**: 对话流中的节点。AI 消息可挂载 `AnalysisResult`。
- **AnalysisResult**: 结构化的分析产出，包含表征分析、内部状态分析及 Markdown 报告。

### 2.3 管理后台 (Admin)
- **Ontology (本体)**: 定义“类”与“属性”的 Schema。
- **KnowledgeBase (知识库)**: 本体的实例化数据集合。
- **SceneTemplate (场景)**: 结合提示词逻辑与知识库的业务入口。

## 3. 核心接口详细说明

### 3.1 证据 (Evidence)
| 字段 | 说明 |
| :--- | :--- |
| evidenceId | 唯一标识，格式如 `EVID_001` |
| type | 来源类型 (dialogue/image/document) |
| content | 原始内容摘要 |
| source | 明确溯源，如文件名、页码、轮次 |

### 3.2 推理链条 (ReasoningChain)
| 字段 | 说明 |
| :--- | :--- |
| triggeredRule | 触发的专家系统规则 |
| conditions | 命中条件的列表，每个条件需回溯 EvidenceId |
| calculatedConfidence | 综合置信度，由各条件置信度加权得出 |

## 4. 建模哲学：原子化心理分析
系统不直接给出感性结论，而是通过：
1. **识别表征 (Observables)**: 观察到了什么？
2. **匹配规则 (Rules)**: 这意味着什么？
3. **输出状态 (Internal States)**: 最终心理画像是什么？

这种三层架构确保了 AI 的输出不是“黑盒”，而是“可解释的透明盒”。
